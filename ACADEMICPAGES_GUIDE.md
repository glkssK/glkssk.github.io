# 如果你想做成和参考网站更接近的 Academic Pages 版本

参考网站底部写明它使用的是：

- Jekyll
- Academic Pages
- GitHub Pages

## 核心思路

1. 直接使用 `academicpages/academicpages.github.io` 模板
2. 仓库命名为 `你的用户名.github.io`
3. 修改 `_config.yml`
4. 替换个人简介、头像、论文、页面导航
5. 用 GitHub Pages 发布

## 适合谁

适合：
- 想长期维护学术主页
- 论文很多，后续还会持续更新
- 可能还想要博客、教学、服务、项目页

不太适合：
- 只想 30 分钟内先把主页上线
- 不想接触 Jekyll / YAML / Markdown

## 我的建议

- 先用这个静态版模板上线
- 等你内容稳定后，再迁移到 Academic Pages

这样最省时间，也最不容易卡在配置问题上。
